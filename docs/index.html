What it does

Collects candidates

The collector calls the read-only social-reader MCP timeline tools and stores compact candidate rows locally.

Protects cursors

It fetches with advance_cursor: false, writes candidates to disk, then calls mark_seen. If marking fails, you get duplicates rather than lost items.

Compiles LLM context

The compiler reads cached candidates, deduplicates them, and emits bounded JSON for a scheduled Hermes digest job.

This repo is intentionally Hermes-host tooling. The MCP server can remain independent and does not need Hermes Agent, digest scripts, SSH access, or shared storage.

Mental model

  1. The MCP server owns social-platform credentials and platform cursor state.
  2. The collector asks the MCP for recent posts without advancing cursors.
  3. The collector writes candidates and batch summaries under the local state directory.
  4. Only after persistence succeeds, the collector calls mark_seen for successful platform/account batches.
  5. The compiler turns cached candidates into bounded JSON that Hermes can summarize.
Failure mode: if the process fails after persistence but before mark_seen, the next run may see duplicates. That is intentional and safer than losing posts.

Quick start

The current local setup uses stdio to run the MCP from the HermesSocialSummerizer checkout.

1. Dry-run the collector

nix run .#collect -- --dry-run --verbose

A dry run fetches and summarizes, but does not write state and does not call mark_seen.

2. Run a real collection

nix run .#collect -- --verbose

3. Compile digest context

nix run .#compile-context -- --since-hours 24 --max-candidates 250

The output is JSON. Feed it into the Hermes LLM job that writes the human-readable daily digest.

State and privacy

Default state directory:

~/.local/state/HermesSocialSummerizer/social-digest/

Override it with:

export SOCIAL_DIGEST_STATE_DIR=/path/to/social-digest-state
PathPurpose
batches/Per-run collection summaries, including per-platform status and mark-seen results.
candidates/YYYY-MM-DD/*.jsonlCompact social candidate rows, one JSONL file per collection batch.
briefings/Reserved for generated briefing artifacts.

Directories are set to 0700; files are set to 0600.

Retention defaults to 30 days and runs only after a successful real collection. Change it with:

hermes-social-digest-collect --prune-days 14

Disable retention pruning with:

hermes-social-digest-collect --prune-days 0

MCP connection

Local stdio mode

export SOCIAL_READER_MCP_TRANSPORT=stdio
export SOCIAL_READER_MCP_COMMAND=/path/to/HermesSocialSummerizer/node_modules/.bin/tsx
export SOCIAL_READER_MCP_ARGS=/path/to/HermesSocialSummerizer/src/server.ts

If an argument contains whitespace, use JSON:

export SOCIAL_READER_MCP_ARGS_JSON='["/path/with spaces/server.ts"]'

HTTP mode for a future LAN MCP

export SOCIAL_READER_MCP_TRANSPORT=http
export SOCIAL_READER_MCP_URL=https://social-reader.lan/mcp
Security note: non-local http:// MCP URLs are blocked unless SOCIAL_READER_MCP_ALLOW_INSECURE_HTTP=true is set. Prefer HTTPS, a localhost tunnel, WireGuard, or an authenticated reverse proxy.

Commands

hermes-social-digest-collect

OptionMeaning
--dry-runFetch and summarize without writing cache or calling mark_seen.
--verbosePrint a run summary as JSON.
--if-previous-hit-limitExit silently unless the previous successful batch hit a fetch cap or incomplete coverage.
--prune-days NDelete state files older than N days after a successful run. Default is 30; 0 disables pruning.

hermes-social-digest-compile-context

OptionMeaning
--since-hours NCandidate window to read from state. Default is 24.
--max-candidates NMaximum deduplicated candidates to emit. Default is 250.

Home Manager installation

The flake exposes a Home Manager module:

hermes-social-digest-pipeline.homeManagerModules.default

Minimal configuration:

{
  imports = [ hermes-social-digest-pipeline.homeManagerModules.default ];

  programs.hermesSocialDigestPipeline = {
    enable = true;
    timers.enable = true;
  };
}

Common options:

programs.hermesSocialDigestPipeline = {
  stateDir = "/home/YOUR_USER/.local/state/HermesSocialSummerizer/social-digest";

  mcp.transport = "stdio";
  mcp.command = "/path/to/HermesSocialSummerizer/node_modules/.bin/tsx";
  mcp.args = [ "/path/to/HermesSocialSummerizer/src/server.ts" ];

  # Whitespace-safe replacement for mcp.args.
  mcp.argsJson = null;

  timers.collectSchedules = [ "10:00" "14:00" "18:00" "22:00" ];
  timers.catchupSchedule = "02:00";
};

When enabled, the module can install the package, link the Hermes skill into ~/.hermes/skills/hermes-social-digest-pipeline, and create user systemd timers.

Recommended daily operation

  1. Run normal collectors at 10:00, 14:00, 18:00, and 22:00.
  2. Run smart overnight catch-up at 02:00 with --if-previous-hit-limit.
  3. At digest time, run the compiler and pass its JSON to the Hermes LLM prompt.
hermes-social-digest-compile-context --since-hours 24 --max-candidates 250

The compiler does not contact the social platforms. It only reads local cached state.

Troubleshooting

SymptomWhat to check
Collector exits before fetchingConfirm SOCIAL_READER_MCP_TRANSPORT, command/path, and MCP server startup.
HTTP MCP URL refusedUse HTTPS, localhost/tunnel, or set SOCIAL_READER_MCP_ALLOW_INSECURE_HTTP=true only on a trusted private network.
Digest context is emptyRun a real collection first, check SOCIAL_DIGEST_STATE_DIR, and verify --since-hours covers the collection window.
Duplicate items appearThis can happen after a failed mark_seen. It is safer than data loss; compiler dedupe should remove many duplicates.
Nix cannot see new filesRun git add -A before nix flake check so flake evaluation sees new tracked paths.

Verification

For development or release checks, run:

nix develop -c npm run typecheck
nix develop -c npm test
nix develop -c npm run build
nix flake check

For an end-to-end non-mutating smoke test:

nix run .#collect -- --dry-run --verbose
nix run .#compile-context -- --since-hours 24 --max-candidates 10
